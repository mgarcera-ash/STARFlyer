01 — Philosophy

Design principles

Clarity over decoration

Every visual element must earn its place. No gradients, shadows, or flourishes for their own sake. Contrast and weight carry hierarchy.

Motion is purposeful

Animations exist to orient, not impress. Entrances confirm arrival. Exits confirm departure. Duration is short — 180–500ms.

Mobile-first, touch-aware

Tap targets are at minimum 44px. Cards have scale feedback on press. Overflow scrolls are touch-friendly and hide scrollbars.

Accessibility is non-negotiable

WCAG 2.2 AA contrast ratios. Semantic roles on all interactive and modal elements. Keyboard navigation throughout.

Dark mode is first-class

Light and dark are treated as equal peers — not afterthoughts. CSS variables are the single source of truth for all color decisions.

Restraint in the palette

The system is near-monochromatic. A single blue (#3b82f6) and green (#10b981) are used for specific semantic roles — not decoration.

Rules

Letter-spacing — Negative tracking (−0.02em to −0.04em) on headings ≥20px. Positive tracking (+0.04–0.12em) on all-caps / eyebrow text only.
Line height — 1.0–1.1 on large display text. 1.3–1.35 on card titles. 1.5–1.65 on body/paragraph text.
Weight contrast — Pair 300/400 (muted/secondary) with 600/700 (primary). Avoid 500 alone — it reads as neither weight.
DM Mono — Reserved for code, hex values, and data fields (hotspot values, form hints). Never used for body copy.

05 — Motion

Animation system

Core easing curve

--ease

cubic-bezier(0.22, 1, 0.36, 1)

An ease-out curve with a fast start and a very gentle, extended deceleration. Feels physical and confident — like something that knows where it's going. Used on all entrance animations.

Keyframe library

fadeScale

Primary entrance. Scale from 0.94→1 while fading in. Used on all content cards and staggered list items.

0.5s --ease · class: .stagger-item
fadeUp

Translates 10px upward while fading in. Used on headers and page-level elements.

0.5s --ease · class: .fade-up
fadeScaleOut

Reverse of fadeScale. Scale from 1→0.94 while fading out. Used for modal exit animations.

0.18s ease
fadeIn / fadeOut

Opacity only. Used for full-screen overlays (About page, search) where scale would be disorienting.

0.25s ease
dot-pulse

Three-dot loading indicator. Each dot cycles 0.2→1→0.2 opacity, staggered by 0.2s.

1.4s ease-in-out infinite · class: .loading-dot
bubble-float

Ambient emoji particles floating upward off-screen. Purely decorative, pointer-events: none.

14s ease-in infinite · class: .bubble-float

Stagger pattern

Base delay — Sections start at their own animationDelay offset (e.g. 0s, 0.08s, 0.16s).
Item stagger — Within a row, each card adds 0.04s × index to the base delay.
Mount-only — Entrance animations use animation-fill-mode: both. They do not replay on re-render.
Exit pattern — Modals use a closing boolean state that plays the exit animation for 180ms before unmounting the component.

07 — Dark Mode

Theme implementation

How it works

Selector — [data-theme="dark"] on <html>. Overrides all CSS variables in one block.
Persistence — localStorage.getItem("theme"). Falls back to prefers-color-scheme media query on first visit.
Toggle — React state (darkMode) drives document.documentElement.setAttribute("data-theme", ...). User-accessible via Quick Links.
No flash — Theme is read from localStorage in a useEffect on mount. A brief flash is acceptable; a <script> block in <head> would eliminate it if required.

Color decisions that differ

--border — Black in light (strong, graphic), #3a3a3c in dark (subtle, receding). The system's most dramatic shift.
--danger — #b91c1c light (dark red for contrast on white), #ef4444 dark (brighter red for contrast on black).
Media card backgrounds — Always #1c1c1e regardless of theme. Image content sets the visual tone.
Blue / Emerald — Fixed across both themes. Sufficient contrast in both contexts.

Rules for adding new elements

Never hardcode a color that appears in both light and dark contexts. Always use a CSS variable.
Elements that are permanently dark (card image layers, featured card) may hardcode dark values — they are theme-stable by design.
When adding a new semantic color, define it in both :root and [data-theme="dark"] before using it.
Test contrast ratios at both themes using WCAG 2.2 AA (4.5:1 normal text, 3:1 large text / UI components).

08 — Accessibility

WCAG 2.2 AA compliance

Contrast ratios

AA Body text on background — black on #f0f0f0 18.1:1
AA Muted text (#737373) on surface (#ffffff) 4.6:1
AA White on blue (#3b82f6) — badge, links 4.5:1+
AA Dark mode: text (#fff) on surface (#1c1c1e) 15.3:1
AA Danger (#b91c1c light / #ef4444 dark) on respective backgrounds ≥4.5:1

Semantic markup rules

Interactive elements — All clickable items are <button> or <a> — never <div onClick>.
Modal dialogs — role="dialog", aria-modal="true", aria-label on the panel element.
Icon-only buttons — Always have aria-label describing the action.
Full-screen sections — role="region" with aria-label. aria-hidden toggled when not visible.
Images — Meaningful alt text on all flyer images.
Lang attribute — lang="en" on <html>.
Minimum touch target — 44×44px on all interactive elements.
Focus management — Search input auto-focuses when overlay opens.
